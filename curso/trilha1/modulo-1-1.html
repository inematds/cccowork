🌐 Cowork vs Chat vs Code

A Anthropic tem três produtos voltados ao usuário final, e a maior parte das pessoas confunde os três. Saber qual usar pra cada coisa é o primeiro passo pra parar de queimar token caro com tarefa errada.

🧭 Os três produtos lado a lado

ProdutoInterfaceFaz ações?Quando usar
Claude (Chat)Web/AppSó responde textoBrainstorm, escrita, análise pontual
Claude CodeCLI (terminal)Sim — arquivos, shell, gitDev, scripts, repositórios
CoworkDesktop AppSim — browser, arquivos, conectoresOperações de negócio, marketing

💡 Dica prática

Dá pra alternar entre Cowork e Code no meio da sessão, preservando contexto. Comece no Cowork (UX amigável); se faltar capacidade ou estourar limite, salte pra Code com um clique.

🎯 Conceitos-chave

  • • Cowork = ambiente agentivo, não chat
  • • Code = mesmo motor, interface CLI
  • • Contexto migra entre os dois
  • • Chat é o "modo leitura"

📊 Onde Cowork brilha

  • • Tarefas que envolvem 3+ ferramentas
  • • Workflows recorrentes (agendados)
  • • Operações com browser real
  • • Decisões agentivas (ramificação)

🤖 Modelo mental: skills como funcionários

Pare de pensar em "prompt gigante". Comece a pensar em equipe. Cada skill é um especialista que só "aparece" quando a tarefa casa com a descrição dele. Sua função não é escrever instruções — é montar e orquestrar o time.

🧱 Skill = funcionário

  • •Nome: identifica o "cargo" (ex: x-insight, contract-review)
  • •Descrição: "currículo". Diz o que ele faz e quando deve ser chamado
  • •Corpo: manual interno. Só é lido quando ele é chamado pra tarefa
  • •Anexos: ferramentas dele (scripts, templates, exemplos)

🔬 Por trás: progressive disclosure

A Anthropic chamou de "progressive disclosure" o jeito como skills são carregadas:

  • Camada 1: só o YAML frontmatter (name + description) está sempre no contexto. Custa pouco token.
  • Camada 2: o corpo do SKILL.md só é carregado quando o Claude decide invocar.
  • Camada 3: anexos (references/, scripts/) só entram quando o corpo pede explicitamente.

Resultado: você pode ter 100+ skills instaladas sem estourar o contexto.

✓ Mentalidade certa

  • ✓"Quem do meu time faz isso?"
  • ✓Skills pequenas e específicas
  • ✓Description clara dispara sozinha
  • ✓Encadear skills resolve o complexo

✗ Mentalidade errada

  • ✗"Vou escrever um prompt-monstro"
  • ✗Skill gigante que faz tudo
  • ✗Description genérica ("ajudar com marketing")
  • ✗Esperar Claude "adivinhar"

💸 Por que Cowork "destruiu" US$ 285 bi

Em janeiro de 2026, o lançamento do Cowork derrubou US$ 285 bilhões em valor de empresas de software num único dia. Não foi pânico de mercado — foi reprecificação racional. Vale entender por quê.

📉 O que aconteceu

  • SaaS de marketing/produtividade: agentes substituem a camada de interface. O dado continua na empresa, mas a ferramenta vira commodity.
  • Agências boutique: tarefas de US$ 10k/mês (research, copy, design, dashboards) viraram task agendada de US$ 50.
  • Camada de orquestração: ferramentas como Zapier perdem valor quando o agente já orquestra nativamente.

🎯 O que isso significa pra você

  • →A janela é agora. Quem aprender Cowork em 2026 fica 6-12 meses na frente.
  • →Operador de agente vira novo "designer de workflow". É papel que paga bem.
  • →Agências que adotam IA agentiva escalam sem contratar. Margens explodem.

🆚 Cowork vs alternativas

Cowork não é a única opção. GPTs da OpenAI, n8n, Make, Zapier — cada um resolve um tipo de problema. Aqui o veredicto honesto sobre quando escolher cada um.

Cowork

Agentivo + Conversa

Brilha em: tarefas com decisão, browser, multi-ferramenta, baixo volume de execuções/dia.

n8n / Make

Determinístico

Brilha em: fluxos repetitivos, alto volume, caminho 100% previsível. Triggers (webhook, cron) maduros.

GPTs (OpenAI)

Instruções + Arquivos

Brilha em: persona/expert customizado, sem ações no mundo real. Bom pra Q&A com base própria.

Zapier

Plumbing

Brilha em: conectar serviço A → B sem código. Ainda útil pra triggers que Cowork não faz nativo.

🧠 Regra de bolso

Conversacional + decisão → Cowork. Repetitivo + alto volume → n8n. Q&A com persona → GPT. Webhook simples → Zapier.

🚀 Seu primeiro chat agentivo

Antes de fechar esse módulo, você vai sentir Cowork na pele. Não com prompt complexo — com algo simples que deixa claro a diferença entre "responder texto" e "executar no mundo".

📝 Prompt do hello world

Oi! Faça o seguinte:

1. Abra o YouTube no browser
2. Procure pelo canal "[seu canal aqui]"
3. Pegue o título dos 3 últimos vídeos
4. Me devolva em formato de tabela com título e data

Antes de executar, confirme o plano comigo.

👀 O que observar

  • 1. Plano: Cowork devolve o plano antes de executar. Ótimo sinal.
  • 2. Browser abre: uma janela do Chrome abre sob controle dele. Você vê o cursor se mover.
  • 3. Erros visuais: se o YouTube mudou layout, ele tenta de novo. Não trava.
  • 4. Output formatado: a tabela vem dentro da conversa, prontinha.

⚠️ Atenção

Não dê acesso a contas sensíveis no hello world. Use canal público. Aprenda a confiar antes de dar permissão de Gmail, Stripe etc.
