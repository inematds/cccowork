🎯 Prompt engineering pra Cowork

Cowork não responde igual chat. Esse módulo te dá a estrutura de prompt que faz o agente acertar de primeira: objetivo declarado, contexto, framework, validação e iteração.

1

🎯 Objetivo declarado primeiro

Esse é o truque que mais melhora resultado. Comece o prompt com "Meu objetivo declarado é...". Cowork usa isso pra resolver ambiguidade em todas as decisões seguintes.

Estrutura

Meu objetivo declarado com este prompt é:
[uma frase clara do que você quer no FIM]

Pra chegar lá, faça:
1. [passo 1]
2. [passo 2]
...

Quero o output em formato: [tabela / markdown / PDF]

💡 Por que funciona

Sem objetivo, o agente otimiza pra "parecer útil". Com objetivo, ele tem norte pra todas as escolhas (qual fonte priorizar, qual formato, quando parar).

2

🧭 Contexto do negócio

Lembra do brand-brief.md que você criou no módulo 1.2? Aqui ele entra em ação. Sempre que possível, referencie seus arquivos de contexto — o output dá um salto de qualidade.

📚 Padrões de referência

  • Use o tom definido em ~/Cowork/ABOUT ME/brand-brief.md
  • Foque no ICP descrito em icp.md
  • Considere meus concorrentes em competitors.md
  • Priorize KPIs listados em kpis.md
3

📐 Frameworks (4 fatores, RACI, RICE)

Quando a tarefa envolve ranquear, comparar ou priorizar, force um framework numerado. Isso obriga o agente a justificar — e você pode auditar.

🧮 Exemplo: matriz 4 fatores (X Insight)

FatorPesoO que mede
Primacy30%Quão fresco é (≤7 dias)
Authority15%Tamanho/credibilidade da fonte
Velocity35%Taxa de crescimento
Content Fit20%Casa com seu nicho

Soma = 100%. Cada signal recebe score 1-5 por fator. Score final 1-20.

4

✋ Confirme antes de executar

Pra tarefas longas (3+ minutos), sempre peça pra confirmar o plano antes de rodar. Salva token e evita refazer.

Padrão

[seu prompt completo]

ANTES de executar, me devolve:
1. Plano em 3-5 passos
2. Quais conectores você vai usar
3. Estimativa de tempo
4. Possíveis pontos de falha

Aí eu confirmo e você executa.

✓ Quando confirmar

  • ✓Tarefa > 3 min
  • ✓Vai escrever em sistema externo (Notion, Gmail)
  • ✓Primeira vez de uma skill nova
  • ✓Ambiguidade ainda no prompt

✗ Quando NÃO precisa

  • ✗Tarefa simples (< 30s)
  • ✗Só leitura, sem efeito colateral
  • ✗Skill já validada que você roda 10x/dia
  • ✗Conversa de brainstorm
5

🔄 Iterar com feedback localizado

Output veio quase certo? Não recomece do zero. Iterar localizando o que mudar economiza tokens e mantém o que estava bom.

Padrão de iteração

Mantém: parágrafos 1, 2 e 4 como estão.
Mudar:
- Parágrafo 3 está genérico demais. Reescreve
  citando o framework Ben (B-E-N: Big, Easy, New)
- Título: troca pra algo com gancho de urgência

Só essas duas mudanças. Não mexa em mais nada.

💡 Dica

Se você diz "refaz tudo", o agente refaz tudo — incluindo o que estava bom. Seja explícito sobre o que preservar.

6

💎 Transformar prompt em skill

Depois de rodar 3-5 vezes com bons resultados, o prompt estabilizou. Hora de cristalizar em skill — você nunca mais reescreve.

🔬 Quando estabilizou?

  • 3+ execuções com mesmo padrão de prompt
  • Output consistente — você não precisa corrigir as mesmas coisas
  • Você sabe quando usar — consegue escrever um "use this skill when..."
  • Existe um padrão de entrada — variáveis claras (nicho, audiência, output)

Prompt pra cristalizar

Esse prompt que rodamos agora — quero virar skill.

Use o skill-creator pra:
1. Extrair as variáveis (o que muda entre execuções)
2. Gerar SKILL.md com name + description forte
3. Sugerir references/ ou scripts/ se fizer sentido
4. Rodar evals comparando com baseline

Quando estiver pronto, salva em ~/Cowork/skills/[nome]/
