🧭 De prompt repetido pra skill

Você sabe que tem uma skill quando roda o mesmo prompt 3-5 vezes com resultado consistente. Antes disso, é template. Depois, vira SKILL.md.

✅ Sinais que a skill estabilizou

  • • Você roda o mesmo "padrão" 3-5 vezes
  • • Output consistente entre execuções
  • • Você sabe descrever quando usar ("use this when X")
  • • Existem variáveis claras (nicho, audiência, output format)
  • • Você corrigiria as mesmas coisas a cada vez

📝 Anatomia do SKILL.md gerado

Esse é o template que skill-creator gera. Estude pra entender — eventualmente você edita à mão.

---
name: weekly-recap
description: Use this skill when the user wants a Friday weekly recap. Triggers on phrases like "weekly recap", "wrap up the week", "what happened this week". Pulls data from Calendar, Gmail, Notion tasks. Outputs 1-page executive summary with wins, frustrations, metrics, next-week priorities.
---

# Weekly Recap Skill

## When to use
- Fridays after 17h
- User explicitly asks for week wrap-up
- End of sprint / cycle

## Inputs needed
- {user_name}: nome pra personalizar
- {week_range}: "Mon-Fri" ou custom
- {focus_areas}: 3-5 áreas que importam pro user

## Process

1. **Pull data sources** (parallel):
   - Google Calendar: eventos da semana
   - Gmail: emails enviados (output) + recebidos importantes
   - Notion: tasks com status changed na semana
   - (optional) Stripe: revenue delta
   - (optional) GitHub: PRs merged

2. **Analyze**:
   - Identify 3 wins (tasks done que importam)
   - Identify 2 frustrations (blocked, recorrente)
   - Calculate métricas que moveram
   - Extract recurring themes

3. **Compose recap** (formato markdown):
   - Header: Week of {date}
   - Wins (bullets, 1-line each, com emoji)
   - Frustrations (bullets, com hipótese de causa)
   - Metrics (table)
   - Next week priorities (top 3, com porquê)

4. **Output**:
   - Save to ~/Cowork/OUTPUTS/recap-YYYY-MM-DD.md
   - Send to Gmail (self)
   - Post summary to Slack #personal (if connected)

## Tone
Direct, honest. Não embeleze frustrações. Não diminua wins.
4

🧪 Rodar evals (validação)

Evals comparam: output COM a skill vs SEM a skill. Se a skill não ganha, descarta. Não dá pra confiar em "achismo".

📊 O que evals medem

  • • Trigger precision: dispara quando deveria? Não dispara quando não deveria?
  • • Quality vs baseline: output melhor que prompt direto?
  • • Token efficiency: resolve em menos passos?
  • • No regression: não quebrou outras skills?
